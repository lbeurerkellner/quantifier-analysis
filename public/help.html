This document describes the high-level features of the instantiation graph editor.

Getting Started

To get started, use the editor on the left to define a set of quantified formulas. For example, you may use the following pair of formulas: 
forall x {f(x)} g(h(x));

forall x {g(x)} f(x);

Based on these formulas, we can now construct an instantiation graph. In the toolbar at the top of the screen, choose the formula you want to start with in the graph construction. Next, press the Create Graph button.

In the graph view on the right, you can now see the intial formula instantiation. The graph includes all instantiated ground terms as well as the assumed ground terms, required to match the trigger patterns of the selected formula.

Terms in the graph which highlighted with a green or red border, allow for further expansion of the instantiation graph (forward and backward steps i.e. further possible formula instantiations). A click on these terms will present a list of possible forward and backward steps.

By application of the offered graph expansion operations, the instantiation graph can be explored to gain a better understanding of the interactions among the provided quantified formulas and their trigger patterns.

User Interface

The user interface comprises of two main components:
  • Formula Editor: In this editor on the left, you can define the formulas to be analysed. For a brief description of the supported syntax see Formula Syntax.
  • Instantiation Graph: In this component, you can construct an instantiation graph based on an initially chosen formula instantiation. For the details of the instantiation graph construction see Graph Construction.
